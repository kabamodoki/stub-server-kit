DEV TOOL

stub-server-kit

ローカル開発用 — 環境変数セットアップ & APIスタブサーバー

① ENV TOOL

環境変数セットアップ

.local.env を読み込んで、本体アプリが参照する環境変数を現在のシェルに一時的に設定します。セッションを閉じると消えます。

② STUB SERVER

APIスタブサーバー

本体アプリが接続する外部API・バックエンドの代役として動作します。YAMLを書くだけで任意のエンドポイントを用意できます。

① env ツール — 使い方

1

.local.env を作成する

env/.local.env.example をプロジェクトルートにコピーして編集します。

# .local.env（例）
API_URL=http://localhost:3132
DB_HOST=localhost
DB_PASSWORD="my password"
NODE_ENV=development
注意 .local.env は Git にコミットしないでください。.gitignore への追加を推奨します。
2

シェルに読み込む

使っているシェルに合わせてコマンドを実行します。

PowerShell
bash / zsh
cmd
# ※ 先頭の ". " (ドット＋スペース) が必須（dot-source）
. .\stub-server-kit\env\load-env.ps1

# ファイルパスを指定する場合
. .\stub-server-kit\env\load-env.ps1 path\to\.local.env
# source または . で実行
source ./stub-server-kit/env/load-env.sh

# ファイルパスを指定する場合
source ./stub-server-kit/env/load-env.sh path/to/.local.env
REM ※ 必ず CALL を付けること（ないと反映されない）
CALL stub-server-kit\env\load-env.bat

REM ファイルパスを指定する場合
CALL stub-server-kit\env\load-env.bat path\.local.env
3

本体アプリを起動する

読み込み後のシェルでそのまま本体アプリを起動すると、設定した環境変数が参照されます。

# 例
yarn start
mvn spring-boot:run
npm run dev

シェルを閉じると環境変数は消えます。永続的な変更は行いません。

② stub サーバー — 使い方

1

依存関係をインストール & 起動する

cd stub-server-kit/stub
mvn spring-boot:run

起動すると http://localhost:3132 で待ち受けます。

2

APIスタブを定義する — config/routes.yaml

YAMLにエンドポイントと返却値を書くだけです。Java不要。

routes:
  # GETエンドポイント
  - method: GET
    path: /api/users
    response:
      status: 200
      body:
        - id: 1
          name: Stub User

  # パスパラメータ（:id）
  - method: GET
    path: /api/users/:id
    response:
      status: 200
      body:
        id: 1
        name: Stub User

  # 遅延レスポンス（ネットワーク遅延の模倣）
  - method: GET
    path: /api/items
    delay: 500
    response:
      status: 200
      body:
        items: []
フィールド 説明 デフォルト
method GET / POST / PUT / PATCH / DELETE GET
path URLパス。:id 形式のパスパラメータ使用可 —
response.status HTTPステータスコード 200
response.body JSONレスポンスボディ（Map / List / 文字列） 空
response.headers 省略可 追加レスポンスヘッダー —
delay 省略可 レスポンス遅延（ミリ秒） 0
再読み込み（再起動不要） routes.yaml を変更したら GET http://localhost:3132/stub/reload にアクセスするだけで反映されます。
3

疑似ログインを使う（認証フロー用）

本体アプリの認証リダイレクト先として stub サーバーを指定します。

# 本体アプリからこの URL へリダイレクトさせる
http://localhost:3132/login?redirect_uri=http://yourapp/callback&state=xyz

ページが表示されたら「ログイン」ボタンを押すと、redirect_uri にトークンを付けてリダイレクトします。

返却パラメータのカスタマイズ

src/main/java/stub/AuthHandler.java の buildRedirectParams() を編集するだけです。

// 認可コードフローに変える例
public Map<String, String> buildRedirectParams(String redirectUri, String state) {
    Map<String, String> params = new LinkedHashMap<>();
    params.put("code", "stub-auth-code-xyz");
    if (state != null) params.put("state", state);
    return params;
}

+ routes.yaml の body はそのまま JSON になる

body: に書いた YAML がそのまま JSON レスポンスとして返ります。特別な変換は不要です。
Content-Type も自動で application/json が付きます。

▶

オブジェクトを返す

routes.yaml
body:
  id: 1
  name: "田中 太郎"
  active: true
レスポンス JSON
{
  "id": 1,
  "name": "田中 太郎",
  "active": true
}
▶

配列を返す

routes.yaml
body:
  - id: 1
    name: "田中"
  - id: 2
    name: "鈴木"
レスポンス JSON
[
  { "id": 1, "name": "田中" },
  { "id": 2, "name": "鈴木" }
]
▶

ネストしたオブジェクトを返す

routes.yaml
body:
  orderId: "ABC-001"
  user:
    id: 1
    name: "田中"
  items:
    - productId: 10
      qty: 2
レスポンス JSON
{
  "orderId": "ABC-001",
  "user": {
    "id": 1,
    "name": "田中"
  },
  "items": [
    { "productId":10, "qty":2 }
  ]
}
▶

body なしでステータスだけ返す（204 No Content 等）

routes.yaml
- method: DELETE
  path: /api/users/:id
  response:
    status: 204
    # body を省略
レスポンス
HTTP 204 No Content
（ボディなし）
YAML の書き方ポイント インデントは半角スペース2つ。タブは使えません。 文字列は基本クォートなしで書けますが、: や # を含む場合はダブルクォートで囲んでください。
